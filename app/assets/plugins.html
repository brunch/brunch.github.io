Plugins

Brunch has a plugin ecosystem to achieve interoperability with various stuff simply via plugin API

For example, if you use JavaScript files in your project, you’ll need a plugin that adds JS support. Same with styles, templates, minifiers, linters and much more.

To install new plugin, simply execute npm install --save plugin-name. This will install its node.js dependencies and save to package.json. Search for brunch on npm for even more plug-ins.

To remove it, edit package.json’s dependencies hash. See npm docs for more docs on packages.

Compilers

Script languages

  • JavaScript-brunch. Provides JS support out-of-box. Without it, your JS files will be ignored.
  • CoffeeScript. CoffeeScript is unfancy JavaScript.
  • IcedCoffeeScript-brunch. IcedCoffeeScript is a CoffeeScript with async syntax.
  • LiveScript-brunch. LiveScript is more functional and feature rich CoffeeScript dialect.
  • EmberScript. EmberScript is Ember tasted CoffeeScript.
  • Roy-brunch. Roy is a small functional language that compiles to js.
  • typescript-brunch. typescript is a typed superset of JavaScript that compiles to plain JavaScript.
  • jsenv-brunch. Add support for compiling environment variables into javascript includes. Useful for Heroku and production/development configuration differences.
  • wisp-brunch. Adds support for Wisp, which is a cute Clojure-like Lisp.
  • react-brunch. Adds support for React templates.
  • es6-module-transpiler-brunch. Adds ES6 module syntax support via es6-module-transpiler.
  • sweet-js-brunch. Adds support for sweet.js.

Style languages

  • CSS-brunch
  • Stylus-brunch. Stylus is robust & powerful style lang. Comes with nib — library of CSS3 cross-browser mixins. See why Stylus is better than LESS / Sass.
  • Sass-brunch. Sass is pretty popular in ruby community. Comes with compass support.
  • LESS-brunch. LESS is the most popular dynamic stylesheet language.
  • Compass Brunch Compiles a project using Compass with support for the configuration file.
  • Rework Brunch Compiles a project using Rework with support for the configuration file.
  • Pleeease-brunch. Pleeease is a css post-processor built with PostCSS. It adds prefixes, packs media queries, minification and provides some polyfills too.
  • PostCSS Brunch Compiles a project using PostCSS (a CSS postprocessor). Add as many PostCSS based polyfills as you want (like Autoprefixer)
  • autoprefixer-brunch. autoprefixer parses plain CSS and takes care of all vendor prefixes, using data from the Can I use... compatibility tables.
  • csscomb-brunch. CSSComb sorts your CSS properties in a specified order and keeps related properties together.
  • stylus-spritesmith-brunch. Adds Stylus support with nib and sprite sheet generation via Spritesmith.

Pre-compiled templates

  • Handlebars-brunch. Handlebars are semantic logic-less templates, compatible with Mustache.
  • Ember-handlebars-brunch. Adds support for pre-compiling Handlebars templates for specific use within Ember.js applications.
  • Eco-brunch. Eco is embedded coffeescript templates that features ERB-like syntax.
  • Mustache (hogan)-brunch. Mustache templates are logic-less and supported in many languages. Hogan is twitter implementation of mustache.
  • Jade-brunch. Jade is a haml big brother with much cleaner syntax.
  • Dust-brunch. Dust is a templating engine designed to provide a clean separation between presentation and logic without sacrificing ease of use.
  • Dust-linkedin-brunch. Dust(LinkedIn) is a fork of the original Dust. LinkedIn will gradually be extending this library with core feature enhancements, helper functions and bug fixes.
  • Jade-angularjs-brunch. Jade to AngularJS module compiler. A la Grunt-angular-templates but for Brunch and Jade.
  • Emblem-brunch. Emblem is a concise, beautiful and compatible templating alternative for handlebars.
  • Markdown-brunch. Markdown allows you to write using an easy-to-read, easy-to-write plain text format, then convert it to structurally valid XHTML (or HTML).
  • Yaml-front-matter-brunch compiles markdown files with YAML front matter into javascript objects with markdown support.
  • Swig-brunch. Swig is a Django-like templating engine for node.js and the browser.
  • Ractive-brunch. RactiveJS is a client side templating and DOM binding engine.
  • nunjucks-brunch. Nunjucks is a rich and powerful templating language for JavaScript.
  • html2js-brunch. Html2JS mimics the functionality of grunt-html2js by converting Angular templates into JavaScript.

Minifiers

  • UglifyJS-brunch. UglifyJS is a popular JavaScript minifier.
  • CleanCSS-brunch. CleanCSS is CSS minifier.
  • Bless-brunch Blesses css files to help out IE browsers with their restrictions for large css files.

Linters

  • JSHint-brunch. JSHint is a community-driven tool to detect errors and potential problems in JavaScript code and to enforce your team's coding conventions.
  • CoffeeLint-brunch. CoffeeLint is a style checker that helps keep CoffeeScript code clean and consistent.

Others

  • Static-jade-brunch — compiles static asset files to html. With it you can get rid of index.html and use index.jade instead.
  • Static-underscore-brunch — same, but for underscore.js.
  • Docco-brunch — adds Docco documentation generator for your app/ directory.
  • JSDoc-brunch — adds JSDoc documentation generator for your app/ directory.
  • Auto-reload-brunch - Adds automatic browser reloading support to brunch.
  • Process-env-brunch - Parses brunch files and replaces the _PROCESS_ENV_FOO token with the value of process.env.FOO.
  • imageoptmizer-brunch — Optimize PNG and JPG images on --optimize. It either uses jpegtran and optipng or node-smushit
  • retina-brunch — Automatically generates non-Retina versions of images in a specified watch folder. Customizable.
  • dependency-brunch — Adds dependOn option to brunch config.
  • keyword-brunch - Adds keyword option to brunch config, replace the keyword after every time compiled.
  • keyword-brunch - An updated version of keyword-brunch, faster, more flexible and available in the npm repository.
  • sprite-brunch — Language agnostic sprite generator for brunch
  • jspreprocess-brunch — Adds C-style preprocessor directive support to JS brunch compilations.
  • groundskeeper-brunch - Clean javascript files (remove console.log ect...) using groundskeeper.
  • gzip-brunch - Adds gzip support to brunch.
  • appcache-brunch - Adds HTML5 .appcache generation to brunch.
  • after-brunch - Execute arbitrary system commands after each compile cycle.
  • brunch-signature - generates a unique signature as part of the build process.
  • digest-brunch - appends a unique SHA digest to asset filenames.
  • git-digest-brunch - cache busts urls using the SHA digest of the current git revision.

  • yaml-i18n-brunch - use yaml to edit i18n translations.
  • constangular-brunch - environment aware YAML configurations for AngularJS.
