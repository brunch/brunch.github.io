Brunch

A lightweight approach to building HTML5 applications with emphasis on elegance and simplicity.

Follow on GitHub

Standing on the shoulders of giants

Instead of reinventing the wheel, brunch assembles awesome wheels
  • MVC framework: Backbone.js
  • Libs: jQuery (or zepto), Underscore
  • Programming languages: CoffeeScript, Roy
  • Templates: Eco, Jade, Mustache, Swig
  • Styles: Stylus, LESS, Sass
Almost everything from the list is optional, you can choose only projects you want. In addition, we have
  • HTML5 Boilerplate normalize.css, helpers.css & index.html
  • Great directory structure
  • A simple express webserver
  • require.js-compatible module system
  • Source code minifiers: uglify.js for javascript and clean-css for stylesheets.
  • textmate / sublime2 bundle
  • growl / libnotify support
Want something else, like HAML support? No problem! We have a nice plugin API and a bunch of third-party plugins.

Getting started

You'll need node.js 0.6. Execute:

npm install -g brunch

You're done!

To create a new project, execute brunch new <project name>.

To build it, execute brunch build in project directory.

To continuously rebuild the project on every change, run brunch watch.

To run a simple http server, execute brunch watch --server.

See docs for more info.

Contributing to Brunch

  • Fork the official repository.
  • Clone your fork: git clone git@github.com:<your-username>/brunch.git
  • Make sure tests are passing for you: npm install && npm test
  • Create a topic branch: git checkout -b features/new-feature-name
  • Add tests and code for your changes.
  • Once you're done, make sure all tests still pass: npm install && npm test
  • Commit and push to your fork.
  • Create an pull request from your branch.
  • Sit back and enjoy.

If the feature you want to add is large, you should probably ask brunch folks if it would be accepted. To do it, start new discussion on github or ping @brunch on twitter.
